Working approach

How the state profiles are built

The profiles combine administrative records, survey indicators, geospatial layers and carefully documented estimates. Each value is reviewed for source year, coverage, comparability and plausibility before it is used in a chart, map or narrative statement.

Evidence workflow

From raw evidence to state profile

The process is designed to be repeatable, transparent and realistic for programme planning rather than a one-off visual exercise.

1

Source

Collect state-level records, survey tables, spatial files and programme documents.

2

Clean

Standardise state names, units, dates, missing values and duplicated records.

3

Score

Convert indicators to comparable 0–100 values and group them by sector.

4

Validate

Check outliers, unrealistic values, source conflicts and year mismatches.

5

Explain

Translate the evidence into map notes, charts, risks and planning messages.

Scoring model

Composite index design

Indicators are not treated equally. Higher-confidence and decision-critical indicators carry more weight, while estimated or older values are used carefully and marked in the profile notes.

Final score = 40% access + 25% economic activity + 20% population pressure + 15% vulnerability

Reality checks

What prevents the profiles from looking artificial

Before publication, each profile is reviewed against known state context. Extreme scores are checked, not automatically accepted.

  • Values are capped where a source creates unrealistic extremes.
  • Older indicators are labelled instead of hidden.
  • State comparisons are shown as relative signals, not absolute truth.
  • Maps and charts are matched with short interpretation notes.

Interpretation principles

How to read the profiles

Comparable, not perfect

The profiles support structured comparison, but they do not replace detailed state-level studies.

Transparent assumptions

Estimated values and data gaps are documented so users understand the confidence level.

Planning focused

The aim is to help teams identify priorities, questions and follow-up analysis areas.
